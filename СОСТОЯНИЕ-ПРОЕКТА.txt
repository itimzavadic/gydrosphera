ГидроСфера — снимок состояния проекта (лендинг)
================================================
Актуализировать этот файл при существенных изменениях сайта или инфраструктуры.

НАЗНАЧЕНИЕ
----------
Одностраничный статический сайт компании «ГидроСфера»: бурение скважин, благоустройство
и ввод по Беларуси. Сборщик не используется: HTML, CSS, JS в каталоге public/.

СТРУКТУРА РЕПОЗИТОРИЯ
---------------------
gydrosphera/
  .gitignore              — служебные файлы (.DS_Store и т.п.)
  serve.sh                — локальный просмотр: поднимает http.server из public/
  public/
    index.html            — разметка и тексты
    css/styles.css        — стили (тёмная тема, неон, шапка, галереи, контакты)
    js/main.js            — меню, лайтбокс, счётчики, карусель благо, свайп галереи
    images/               — фото и графика (logo.jpeg, main1.jpg, main.jpeg, blago_*, …)
    robots.txt            — Allow: /, Sitemap: https://gydrosphera.by/sitemap.xml
    sitemap.xml           — одна страница: https://gydrosphera.by/

Деплой: GitHub (репозиторий проекта) + Vercel; корень сайта на хостинге — папка public/.
После push в ветку main Vercel обычно пересобирает сайт автоматически.

ПУБЛИЧНЫЙ ДОМЕН (на момент описания)
------------------------------------
gydrosphera.by (HTTPS). Проверка Яндекс.Вебмастер: meta name="yandex-verification"
в index.html. Отдельный HTML-файл верификации Яндекса удалён — используется только мета-тег.

ЗАПУСК ЛОКАЛЬНО
----------------
  ./serve.sh
или
  cd public && python3 -m http.server 8080
Открыть http://127.0.0.1:8080 (не file://).
